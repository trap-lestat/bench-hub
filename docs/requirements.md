# 需求说明

## 1. 项目目标
- 提供可扩展的 Go 后端 API，用于个人项目核心业务能力。
- 使用 Locust 构建可复用的压测方案，验证性能基线与容量。
- 提供 Vue 管理后台用于配置与数据查看。

## 2. 核心功能（MVP）
- 用户认证：登录、刷新令牌、退出（JWT）。
- 资源管理：基础 CRUD（示例实体可后续替换）。
- 列表查询：分页、排序、过滤。
- 系统健康检查：/health、/metrics。
- 管理后台：登录、列表页、详情页、创建/编辑。

## 3. 非功能需求
- 性能：
  - 95% 请求延迟 < 300ms（API 读操作）
  - 错误率 < 1%
- 稳定性：支持长时间压测（>= 30 分钟）。
- 可维护性：统一日志、错误码、配置管理。
- 可观测性：基本指标与链路追踪字段。

## 4. 约束与假设
- 单体服务优先，后续可拆分。
- 数据库优先选择 PostgreSQL。
- 仅开发最小可用管理后台。

## 5. 里程碑（初稿）
- M1：后端骨架 + 基础 API 文档
- M2：管理后台 MVP
- M3：Locust 压测方案 + 报告

## 6. 验收标准
- API 通过基本集成测试与手测。
- Locust 脚本可稳定复用并输出报告。
- 前后端主要功能闭环可用。
